什么是代币？

  • Alt Coin，也即alternative coin。代表着该币是从比特币的设计思想或者源码产生，甚至直接硬分叉而来的，与比特币相似的货币。
  • 非正式定义
  • from: Sia白皮书

什么是two-way peg？

  • 双向锚定。是指使数字货币得以在主链与侧链之间（甚至不同货币系统之间）流转，通常通过冻结与反向冻结实现。
  • 非正式定义
  • from：Sia白皮书

RS code

  • 也就是Reed-Solomon编码。
  • 如果把Reed-Solomon编码当作黑盒的话，只需要知道它是可以使用(M,N)-Scheme的编码种类（N>M）。也就是说，把原始数据分为N片，其中M片是原始数据切片，N-M片是冗余编码。在解码的时候，只需要获取N片任意其中M片，即可正确恢复原始数据。

什么是硬分叉？什么是软分叉？

  • 无论是哪种分叉，一般都是由于不同节点之间共识不同导致的。
  • 不太严谨的说：
    • 软分叉是指，即使共识规则发生了改变，但是未升级部分的节点与已升级部分的节点在遵守共识规则的情况下能够彼此验证。任一个产生的块都会被所有城市节点，无论是否升级，接受或者拒绝。不会分散算力，不会有诚实节点产生分歧的情况。
      • 下图展示一种软分叉的情况。
    • 硬分叉一般是由于升级的节点与未升级节点之间产生的块不能彼此验证。可能是已升级的节点认为未升级节点的块是无效块，也可能是未升级的节点认为已升级的节点产生的块是无效块。总之，算力被分散。不被某个节点集合认可的一部分块被另一个节点集合认可，因此这部分节点集合认可的链出现不同。（这是因为每个节点只相信自己看到的，在这些节点眼里，这部分的块有效。
      • 在下图中，展示硬分叉的一种情况。未升级的节点不认为已升级节点产生的块有效，因此拒绝将之添加到自己维护的链上；但是已升级节点却可以兼容未升级的块的格式，因此认为他有效。于是不同的节点集合维护了不同的链。
